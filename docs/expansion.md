# Expansion Guidelines

## Adding New Features
- Create a new module/file in the relevant directory (`core`, `api`, `plugins`, `utils`).
- Document new features in `docs/features.md` and update `docs/file_structure.md` if structure changes.

## Adding Plugins
- Place plugin code in `src/plugins/` or a dedicated `plugins/` directory.
- Register new plugins in the config file.

## Adding Documentation
- Add or update Markdown files in `docs/` for every major feature or change.

## Best Practices
- Keep modules small and focused.
- Use clear, descriptive names for files and functions.
- Update documentation with every code change.
