# Maintenance

## Keeping the Project Healthy
- Regularly update dependencies (see `Cargo.toml`).
- Refactor code as needed to maintain clarity and performance.
- Run tests before merging changes (testing system planned).
- Keep documentation up to date.

## For New Developers
- Read all docs in the `docs/` folder before starting.
- Ask questions via issues or discussions if anything is unclear.

## For Returning Developers
- Review `docs/roadmap.md` and `docs/features.md` for project status.
- Check for dependency or Rust edition updates.
